总体开发须知

  • 1、字符串中使用单引号 '，JSX 中使用双引号 "
  • 2、采用两格空格缩进
  • 3、方法体控制在一屏内显示 (30行左右)
  • 4、直接使用 this，不要使用 that/self/me。
  • 5、代码具有连续性，避免无意义的换行
  • 6、用 es5-shim 和 es6-shim 里的方法，而不是lodash或jquery的同名方法，如 forEach, bind
  • 7、回调原则上不允许超过两层，三层必须用promise处理，建议使用 async await
  • 8、pages 、component命令方式一率用大驼峰
  • 9、统一使用unix换行符，即/n
  • 10、不要使用 getter/setter，使用明确的 method
  • 11、使用 === 和 !== 替换 == 和 !=，只有在判断 undefined 和 null 时除外，用 undefOrNull == null
  • 12、顶层函数定义使用 function name(param1) {}，不要使用 arrow function
  • 13、文件头部需要添加作者信息，以便于快速联系，author 可以是一个也可以是多个，格式如下：
/**
 * @file 表格显示组件（简要描述下文件功能和应用场景）
 * @author liujintao
 */
  • 14、使用解构
// bad
const splitLocale = locale.split('-');
const language = splitLocale[0];
const country = splitLocale[1];

// good
const [language, country] = locale.split('-');
  • 15、eslint noConsoleLogRule (禁止提交 console.log 代码)
  • 16、noDeadCodeRule (检测无用的注释和 dead code)
  • 17、noInlineStyleRule (禁止使用内联样式)
  • 18、methodLengthRule (限制方法的行数)
  • 19、 具体使用情况后续调整修复

TS 使用规范

1、组件的 props 要写清楚类型，function 回调需要精确到参数类型

2、对于接收可渲染类型的 props，不要只使用 string，应该使用 React.ReactNode，以便支持传入组件、dom、number、string等

3、Reducer State 使用规范：

export class State {
  cnt = new AsyncTuple<number>();
  projectEnName = '';
  projectCnName = '';
}

const initialState = new State();
function Reducer(state: State, action: Action): State {
  const { payload, type } = action;
  switch (type) {
    case Types.loading: {
      return ...
    }
  }
}

4、combineReducers 加上注释:

const reducer = combineReducers({
   /** 导航 */
   nav: NavReducer,
   /** 搜索  */
   search: SearchReducer
});

5、 paramTypeRule (声明参数类型)

6、VerboseTypeRule (检测多余的类型声明，可以根据初始值推导出来的无需再声明类型)

7、 mapKeyRule (迭代对象需要添加 "Key" 属性)

Git Commit 建议

完整的规范如下

<type>(<修改内容范围>):<一句话说明改了什么>
<空行>;
<内容详情，可以写多行>

简单规范

<type>(<修改内容范围>):<一句话说明改了什么>
<type>:<一句话说明改了什么>

示例：

feature(Button): 添加自动转账按钮

我是一个神奇的按钮，点一下给刘金涛转100W。
点二下转200W。

type 可选值：

  • feat: 新功能
  • fix: 修复问题
  • docs: 文档修改
  • format: 修改代码格式，不影响代码逻辑
  • clean: 清理代码，移除过期代码
  • refactor: 重构代码，非添加新功能，非修复漏洞
  • perf: 性能提升
  • test: 添加测试
  • chore: 修改工具相关（不限于文档、代码生成，Makefile 修改）
  • deps: 依赖更新
