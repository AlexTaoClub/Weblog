继承方式

原型链继承

function Parent() {
    this.val = 18;
    this.arr = [1];
}
function Child() {

}
Child.prototype = new Parent();              // 核心

var child_1 = new Child();
var child_2 = new Child();

child_1.val = 20;
child_1.arr.push(2);

console.log(child_1.val); // 20
console.log(child_2.val); // 18
console.log(child_1.arr); // 1, 2
console.log(child_2.arr); // 1, 2

核心： 父类实例充当子类原型对象

优点： 简单，容易

缺点： 1.修改child_1.arr后child_2.arr也变了，因为来自原型对象的引用属性是所有实例共享的。

​ 2.创建子类实例时，无法向父类构造函数传参

借用构造函数继承

function Parent(val) {
    this.val = val;
    this.arr = [1];
    this.fun = function() {
        // ...
    }
}
function Child(val) {
    Parent.call(this, val);       // 核心
    // ...
}

var child_1 = new Child(1);
var child_2 = new Child(2);
child_1.arr.push(2);

console.log(child_1.val); // 1
console.log(child_2.val); // 2
console.log(child_1.arr); // 1, 2
console.log(child_2.arr); // 1

核心： 借父类的构造函数来增强子类实例，等于是把父类的实例属性复制了一份给子类实例装上了（完全没有用到原型）

优点： 1.解决了子类实例共享父类引用属性的问题

​ 2.创建子类实例时，可以向父类构造函数传参

缺点： 1.无法实现函数复用，每个子类实例都持有一个新的fun函数，太多了就会影响性能，内存爆炸。

组合继承 （常用）

function Parent(name) {
    this.name = name
};
Parent.prototype.sayname = function() {
    console.log(this.name)
};


function Child(arg) {
    Parent.call(this, arg);                // 核心
};
Child.prototype = new Parent();            // 核心

var child = new Child("child")
var parent = new Parent("AlexTao");

console.log(parent.sayname())    //AlexTao
console.log(child.sayname())     //child
child instanceof Parent          // true

核心： 把实例函数都放在原型对象上，以实现函数复用。

​ 通过Parent.call(this)---继承父类的基本属性和引用属性并保留能传参的优点；

​ 通过Child.prototype = new Parent()---继承父类函数，实现函数复用

优点： 1.不存在属性共享问题

​ 2.可传参

​ 3.函数可复用

缺点： 1.子类原型上有一份多余的父类实例属性 。内存浪费 。

寄生组合继承（最佳）

function Parent(value) {
    this.val = value
}
Parent.prototype.getValue = function() {
    console.log(this.val)
}

function Child(value) {
    Parent.call(this, value)                                  // 核心
}
Child.prototype = Object.create(Parent.prototype, {           // 核心
    constructor: {
        value: Child,
        enumerable: false,
        writable: true,
        configurable: true
    }
})
const child = new Child(18)

child.getValue() // 18
console.log(child instanceof Parent) // true

核心： 将父类的原型赋值给了子类，并且将构造函数设置为子类。

优点： 1.解决了无用的父类属性问题，还能正确的找到子类的构造函数。

Class继承

class Parent {
    constructor(value) {
        this.val = value
    }
    getValue() {
        console.log(this.val)
    }
}
class Child extends Parent {
    constructor(value) {
        super(value)           // 关键
        this.val = value
    }
}
let child = new Child(18)
child.getValue() // 18
console.log(child instanceof Parent) // true

核心： 使用 extends 表明继承自哪个父类，并且在子类构造函数中必须调用 super，因为这段代码可以看 成 Parent.call(this, value)。
